Grid Layout System

A responsive CSS grid system for arranging cards in various sizes while maintaining proper spacing and proportions.

Key features:

  • 1x1 cards are always square (aspect-ratio: 1/1)
  • Other card layouts (2x1, 1x2, 2x2) fill the appropriate number of grid squares plus gaps
  • Responsive breakpoints adjust grid size for different screen sizes
  • Consistent gap spacing between all grid items

Implementation Example

<section class="grid-container">
  <article class="card card-1x1">...</article>
  <article class="card card-2x1">...</article>
  <article class="card card-1x2">...</article>
  <article class="card card-2x2">...</article>
</section>

CSS Implementation

/* Grid Layout Container */
.grid-container {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(150px, 1fr));
  gap: 1rem;
  width: 100%;
}

/* Responsive grid adjustments */
@media (min-width: 576px) {
  .grid-container {
    grid-template-columns: repeat(auto-fill, minmax(180px, 1fr));
  }
}

@media (min-width: 768px) {
  .grid-container {
    grid-template-columns: repeat(auto-fill, minmax(196px, 1fr));
  }
}

@media (min-width: 992px) {
  .grid-container {
    grid-template-columns: repeat(auto-fill, minmax(256px, 1fr));
  }
}

/* Card Size Classes */
.card-1x1 {
  grid-column: span 1;
  grid-row: span 1;
  aspect-ratio: 1/1;
}

.card-2x1 {
  grid-column: span 2;
  grid-row: span 1;
}

.card-1x2 {
  grid-column: span 1;
  grid-row: span 2;
}

.card-2x2 {
  grid-column: span 2;
  grid-row: span 2;
}

/* Scoped card styles to avoid breaking other layouts */
.grid-container > article.card {
  overflow: hidden;
  width: 100% !important;
  height: 100% !important;
  flex-shrink: 0;
}

When using the grid layout:

  1. Add the .grid-container class to the parent container
  2. Add the appropriate size class (card-1x1, card-2x1, etc.) to each card
  3. For image cards, ensure images use object-fit-cover for proper scaling

Important Implementation Notes

The grid layout CSS has been carefully scoped to avoid conflicts with other layouts:

  • Card styles within grid containers use .grid-container > article.card selectors
  • Non-grid layouts use section:not(.grid-container) > article.card selectors
  • This separation ensures the grid layout doesn't break existing card layouts

For non-grid cards, the following constraints are applied:

  • Maximum height matches the size of a 1x1 card
  • Maximum width is 2x the width of a 1x1 card
  • These dimensions are responsive using CSS variables
/* CSS variables for responsive card sizing */
:root {
  --card-base-size: 150px; /* Mobile default */
}

@media (min-width: 576px) {
  :root { --card-base-size: 180px; }
}

@media (min-width: 768px) {
  :root { --card-base-size: 196px; }
}

@media (min-width: 992px) {
  :root { --card-base-size: 256px; }
}

/* Base card sizing for all cards */
.card {
  height: var(--card-base-size);
  width: var(--card-base-size);
}

/* Non-grid card sizing */
section:not(.grid-container) > article.card {
  height: var(--card-base-size);
  width: var(--card-base-size);
}

/* Special card types */
.card-2x1 {
  width: calc(var(--card-base-size) * 2 + 1rem);
}

.card-2x2 {
  width: calc(var(--card-base-size) * 2 + 1rem);
  height: calc(var(--card-base-size) * 2 + 1rem);
}

If you experience any layout issues, check that your selectors are properly scoped.
